## What Is This?

The Soteria Model is an open-source framework for building community-based mutual aid and protection networks. It provides organizational structures, security practices, and operational templates that any community can adapt to their local context.

**This is not software** â€” it's documentation. A field guide. A blueprint that communities can use to organize themselves effectively and safely.

## Key Features

- **Compartmentalized Structure**: Cells, segments, and stewards designed for security
- **Web of Trust**: Vouching-based membership that resists infiltration  
- **Federated & Autonomous**: No central authority; each community is independent
- **Multi-language**: Available in English, Spanish, Somali, and Arabic
- **Printable Materials**: Rights cards, flyers, and community resources
- **Privacy by Design**: No tracking, no analytics, no data collection

## Who Is This For?

- **Communities** seeking to organize mutual aid networks
- **Organizers** building protection infrastructure
- **Anyone** who wants to adapt this model for their local context

## Quick Start

### View the Documentation

Visit [soteriamodel.com](https://soteriamodel.com) to read the full documentation.

### Run Locally

```bash
# Clone the repository
git clone https://github.com/mkokes/soteria-model.git
cd soteria-model/docs

# Install dependencies
npm install

# Start development server
npm run dev

# Build for production
npm run build
```

### Fork & Adapt

1. Fork this repository
2. Modify content for your community's needs
3. Deploy to GitHub Pages, Netlify, or any static host
4. Remove any references to the original if desired

## Contributing

This project welcomes contributions:

- **Translations**: Help translate content to additional languages
- **Content improvements**: Clarify language, fix errors, improve accessibility
- **Local adaptations**: Share how you've adapted the model (without operational details)

Please do **not** contribute:
- Operational details from any real network
- Member information or identities
- Location-specific sensitive information

## Principles

1. **Compartmentalization** â€” Segments don't overlap; information doesn't flow freely
2. **Vouching Over Recruitment** â€” Every member is personally vouched for
3. **Autonomy** â€” Each cell operates independently
4. **Privacy by Design** â€” Collect nothing unnecessary
5. **Replicability** â€” This model is designed to be copied
6. **Solidarity Over Charity** â€” Neighbors protecting neighbors
7. **Harm Reduction** â€” Meet people where they are
