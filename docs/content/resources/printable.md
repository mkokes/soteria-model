## Checklists

### Founding Team Checklist

Use this when starting a new cell with 3-5 trusted people.

```
FOUNDING TEAM STARTUP CHECKLIST

Before Launch:
[ ] 3-5 founders identified and committed
[ ] All founders deeply trust each other
[ ] Core Principles reviewed and agreed
[ ] Security training completed together
[ ] Signal infrastructure set up
[ ] Steward roles assigned
[ ] Decision-making process agreed
[ ] Meeting schedule established

First Month:
[ ] Initial segments defined
[ ] Onboarding process documented
[ ] First members identified for vouching
[ ] First activities planned
[ ] Emergency protocols established
```

---

### New Member Onboarding Checklist

Use this for every new member joining your cell.

```
ONBOARDING CHECKLIST

Vouching:
[ ] Voucher confirms they trust prospect
[ ] Onboarding Steward notified
[ ] Initial conversation scheduled

Initial Conversation:
[ ] Steward, voucher, and prospect meet
[ ] Cell explained, questions answered
[ ] Commitment and expectations clear
[ ] Decision made to proceed (or not)

Security Training:
[ ] Signal installed and configured
[ ] Screen Lock enabled
[ ] Registration Lock enabled
[ ] Disappearing messages explained
[ ] Device security reviewed
[ ] Compartmentalization explained
[ ] What NOT to share reviewed
[ ] Compromise protocol reviewed

Probation Start:
[ ] Added to one segment group
[ ] Voucher and segment lead notified
[ ] First activities assigned
[ ] Check-in schedule set

Probation Complete:
[ ] Segment lead assessment
[ ] Voucher feedback
[ ] Trusted status confirmed
[ ] Full segment access granted
```

---

### Security Audit Checklist

Conduct this audit monthly with all members.

```
MONTHLY SECURITY AUDIT

For All Members:
[ ] Signal still configured correctly
[ ] Phone lock enabled
[ ] Registration lock enabled
[ ] Disappearing messages active
[ ] No sensitive info in screenshots
[ ] No operational details in other apps

For Segment Leads:
[ ] All members following compartmentalization
[ ] No cross-segment information sharing
[ ] Communication channels secure
[ ] No new security concerns

For Stewards:
[ ] Vouching process being followed
[ ] No trust concerns with any members
[ ] Emergency protocols still current
[ ] Backup communication tested
```

## Printing Guidelines

### For Rights Cards
- Print on cardstock for durability
- Cut along dotted lines
- Fold to wallet size
- Laminate if possible

### For Checklists
- Print on regular paper
- Use one per member/activity
- Destroy after completion
- Never store digitally

### For Role Cards
- Print on cardstock
- Give to members during onboarding
- Replace if lost (don't keep copies)

---

::: warning SECURITY REMINDER
**Never store completed forms digitally.** Keep physical copies in a secure location. Destroy forms when members leave the cell.
:::
