# Plantillas

Plantillas y listas de verificación descargables para ayudarte a iniciar y operar tu célula.

::: tip PERSONALIZA
Estos son puntos de partida. Adáptalos a las necesidades, idioma y contexto de tu comunidad.
:::

## Listas de Verificación

### Lista de Verificación del Equipo Fundador

```
LISTA DE VERIFICACIÓN DE INICIO DEL EQUIPO FUNDADOR

Antes del Lanzamiento:
[ ] 3-5 fundadores identificados y comprometidos
[ ] Todos los fundadores confían profundamente entre sí
[ ] Principios Fundamentales revisados y acordados
[ ] Entrenamiento de seguridad completado juntos
[ ] Infraestructura de Signal configurada
[ ] Roles de guía asignados
[ ] Proceso de toma de decisiones acordado
[ ] Calendario de reuniones establecido

Primer Mes:
[ ] Segmentos iniciales definidos
[ ] Proceso de incorporación documentado
[ ] Primeros miembros identificados para aval
[ ] Primeras actividades planificadas
[ ] Protocolos de emergencia establecidos
```

### Lista de Verificación de Incorporación de Nuevos Miembros

```
LISTA DE VERIFICACIÓN DE INCORPORACIÓN

Aval:
[ ] Avalador confirma que confía en el prospecto
[ ] Guía de Incorporación notificado
[ ] Conversación inicial programada

Conversación Inicial:
[ ] Guía, avalador y prospecto se reúnen
[ ] Célula explicada, preguntas respondidas
[ ] Compromiso y expectativas claros
[ ] Decisión tomada de proceder (o no)

Entrenamiento de Seguridad:
[ ] Signal instalado y configurado
[ ] Bloqueo de Pantalla habilitado
[ ] Bloqueo de Registro habilitado
[ ] Mensajes que desaparecen explicados
[ ] Seguridad de dispositivo revisada
[ ] Compartimentación explicada
[ ] Qué NO compartir revisado
[ ] Protocolo de compromiso revisado

Inicio de Probación:
[ ] Añadido a un grupo de segmento
[ ] Avalador y líder de segmento notificados
[ ] Primeras actividades asignadas
[ ] Calendario de check-ins establecido

Probación Completa:
[ ] Evaluación del líder de segmento
[ ] Retroalimentación del avalador
[ ] Estatus de confianza confirmado
[ ] Acceso completo al segmento otorgado
```

### Lista de Verificación de Auditoría de Seguridad

```
AUDITORÍA DE SEGURIDAD MENSUAL

Para Todos los Miembros:
[ ] Signal sigue configurado correctamente
[ ] Bloqueo de teléfono habilitado
[ ] Sin datos innecesarios almacenados
[ ] Mensajes que desaparecen activos en todos los grupos

Para Guías:
[ ] Membresía de grupos revisada
[ ] Antiguos miembros removidos
[ ] Lista de admins actualizada
[ ] Sin historial de mensajes innecesario

Para la Célula:
[ ] ¿Alguna preocupación de seguridad del mes pasado?
[ ] ¿Alguien necesita re-entrenamiento?
[ ] ¿Se necesitan mejoras de proceso?
[ ] ¿Protocolo de compromiso conocido por todos?
```

## Tarjetas de Rol

### Tarjeta de Rol de Líder de Segmento

```
LÍDER DE SEGMENTO

Responsabilidades:
• Manejar grupo de Signal del segmento
• Coordinar actividades del segmento
• Incorporar nuevos miembros al segmento (después de verificación)
• Asegurar que se sigan prácticas de seguridad
• Reportar capacidad al Coordinador de Logística

Conoces:
• Los miembros de tu segmento
• Otros líderes de segmento
• Coordinador de Logística

No Conoces:
• Miembros de otros segmentos
• Detalles operativos de otros segmentos
• Ubicaciones de individuos vulnerables

Recordatorios de Seguridad:
• Se requiere la seguridad de dispositivo más fuerte
• Revisión regular de membresía del grupo
• Plan de sucesión en su lugar
```

## Protocolos de Emergencia

### Si Alguien es Detenido

```
RESPUESTA A DETENCIÓN

Inmediatamente:
1. Confirmar detención (quién, cuándo, dónde, por quién)
2. Alertar al segmento de Observadores Legales
3. Contactar abogado de inmigración
4. Alertar contacto de emergencia familiar

NO:
• Publicar en redes sociales
• Discutir detalles en grupos grandes
• Especular sobre razones

Documentar:
• Hora y ubicación
• Agencia involucrada
• Nombres/números de placa si se conocen
• Testigos
```

### Si Se Sospecha Compromiso

```
PROTOCOLO DE COMPROMISO

Si sospechas una brecha de seguridad:

1. PAUSAR todas las actividades
2. ALERTAR a los Guías por canal separado
3. EVALUAR qué información puede estar comprometida
4. CONTENER — ¿quién necesita saber?
5. DECIDIR respuesta basada en severidad

Niveles:
• Dispositivo perdido/robado → Miembro cambia cuentas, removido de grupos
• Miembro comprometido → Revocar acceso, evaluar exposición
• Segmento comprometido → Suspender segmento, evaluar
• Amenaza activa → Suspensión total, solo coordinación en persona
```
