# التوسع

نموذج سوتيريا مصمم للتوسع — داخل الخلية وعبر المجتمعات. التوسع دائماً **اختياري** و**يُقرر محلياً**.

## التوسع داخل الخلية

مع نمو الخلية، قد تحتاج لتوزيع عبء العمل:

### إضافة مشرفين مشاركين

عندما يكون مشرف واحد مثقلاً:

| العلامة | الحل |
|---------|------|
| أوقات الاستجابة تتباطأ | إضافة مشرف مشارك لتقاسم الحمل |
| نقطة فشل واحدة | تدريب مشرف احتياطي |
| خطر الإنهاك | تدوير المسؤوليات |

### تقسيم الأقسام

عندما ينمو القسم فوق ~15 عضواً، فكر في:

- **تقسيم جغرافي**: شرق/غرب، شمال/جنوب، حسب الحي
- **تقسيم وظيفي**: السائقون مقابل المرسلين، الفارزون مقابل الموزعين
- **تقسيم زمني**: فريق النهار مقابل فريق المساء

كل قسم فرعي يحصل على مجموعة Signal وقائد خاصين به.

### منسقو المناطق

للخلايا التي تغطي مناطق واسعة:

```
┌─────────────────────────────────────────────────────────┐
│                   خلية المدينة                           │
│                                                          │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐      │
│  │   الشمال   │  │   الوسط    │  │   الجنوب   │      │
│  │   المنطقة  │  │   المنطقة  │  │   المنطقة  │      │
│  └──────┬──────┘  └──────┬──────┘  └──────┬──────┘      │
│         │                │                │              │
│         └────────────────┼────────────────┘              │
│                          │                               │
│                 ┌────────▼────────┐                      │
│                 │ قادة المناطق   │                      │
│                 │    التنسيق     │                      │
│                 └─────────────────┘                      │
└─────────────────────────────────────────────────────────┘
```

منسقو المناطق يديرون اللوجستيات لمنطقتهم مع الحفاظ على ممارسات الأمان على مستوى الخلية.

## متى لا تتوسع

التوسع اختياري. بعض السياقات تعمل أفضل بالبقاء صغيرة:

| الوضع | التوصية |
|-------|---------|
| بلدة صغيرة/نائية | خلية واحدة قد تخدم إلى أجل غير مسمى |
| مستوى نشاط منخفض | لا تنشئ هيكلاً لا تحتاجه |
| شبكة الثقة لا تزال تتطور | نمِّ الثقة قبل نمو الهيكل |
| الضغط الخارجي يتزايد | توحد بدلاً من التوسع |

**الأصغر قد يكون أكثر أماناً.** المزيد من الهيكل يعني المزيد من التعقيد والمزيد من نقاط الفشل المحتملة.

## إنتاج خلايا جديدة

عندما يريد مجتمع جديد البدء:

### طريقة المجموعة البذرة

1. **تحديد المؤسسين**: 3-5 أشخاص من المنطقة الجديدة، مزكون من الشبكة الحالية
2. **التدريب**: المؤسسون يتلقون تدريب كامل بمستوى المشرفين
3. **الاستقلالية**: الخلية الجديدة تعمل مستقلة من اليوم الأول
4. **لا تسلسل هرمي**: الخلية الأصلية ليس لها سلطة على الخلية الجديدة

### مشاركة المعرفة (اختياري)

الخلايا قد تختار مشاركة:
- معلومات قانونية محدثة
- تحسينات ممارسات الأمان
- تحسينات القوالب
- الدروس العامة المستفادة

الخلايا يجب ألا تشارك:
- قوائم الأعضاء
- التفاصيل التشغيلية
- قنوات الاتصال
- تفاصيل الهيكل الداخلي

### نموذج الاتصال (اختياري)

إذا أرادت الخلايا اتصالاً مستمراً:

```
┌──────────────┐          ┌──────────────┐
│   الخلية أ   │          │   الخلية ب   │
│              │          │              │
│  ┌────────┐  │          │  ┌────────┐  │
│  │ الرابط │◄─┼──────────┼─►│ الرابط │  │
│  └────────┘  │          │  └────────┘  │
└──────────────┘          └──────────────┘
```

- شخص واحد معين لكل خلية للاتصال بين الخلايا
- يشارك فقط المعلومات غير التشغيلية
- لا يمكنه إلزام خليته بأي شيء

## مبادئ التوسع

1. **توسع عند الحاجة، ليس عند الإمكان** — لا تنشئ هيكلاً لذاته
2. **القرارات المحلية** — كل خلية/قسم يقرر متى وكيف يتوسع
3. **الحفاظ على التجزئة** — التوسع يجب ألا يكسر حدود الأمان
4. **وثق مع نموك** — الهياكل الأكبر تحتاج عمليات أوضح
5. **الاستقلالية فوق الكفاءة** — الخلايا المستقلة أكثر أماناً من المنسقة
